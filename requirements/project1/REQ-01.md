# Parallel strategy creation

**ID:** REQ-01

**Type:** Functional

**Originator:** Stakeholder / Client Interview

**Description:**
The system shall allow a user to create and maintain multiple trading strategies concurrently.

**Rationale:**
Users need to work on several strategy ideas in parallel without being forced to close or archive one before starting another.

**Fit Criterion:**

- At least two independent strategies can be configured simultaneously.
- No data or state interference occurs between concurrently open strategies.

**Customer Satisfaction (0–5):**

- 3: Enables efficient multi-strategy workflows without context switching loss.

**Customer Dissatisfaction (0–5):**

- 1: Users can work around by working sequentially, but productivity is significantly reduced.
